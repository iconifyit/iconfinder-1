Building Mobile Applications with OpenUI5

Topics specific to developing applications in the sap.m library

Note sap.m doesn't provide any support in regards to accessibility mode except for first steps in keyboard handling.
In this section:
  • Handling Navigation and Lifecycle Events
    Mobile apps are often composed of several pages and the user can drill-down to detail pages and go back up again. This is often visualized by horizontal slide animations. OpenUI5 supports this pattern by providing the sap.m.App and sap.m.NavContainer controls, which handle the navigation between the pages.
  • Adapting to Platform and Form Factors
    The OpenUI5 Mobile library target devices run different operating systems and have very different screen sizes and pixel densities. OpenUI5 does some adaptations automatically. In addition to that, the application can further adapt to the current device.
  • Working with Lists and Tables
    sap.m.List and sap.m.Table both inherit from the abstract sap.m.ListBase and provide the features used for lists and tables.
  • Working with Facet Filters
    Facet filters support users in finding the information they need from potentially very large data sets.
  • Triggering Phone, SMS and E-Mail
    With OpenUI5 for Mobile you can easily trigger native mobile phone applications such as e-mail, telephone, and text messages.
  • Scrolling in OpenUI5 Mobile
    Because of limited size of mobile devices, scrolling is an essential topic in mobile user experience. Smooth and easy scrolling is important for user acceptance of mobile applications.
  • Running Mobile Apps in Hybrid Web Containers
    SAPUI5 supports two options for building mobile apps: As a web application loaded from a URL, or as a hybrid app consisting of a native app wrapper, for example PhoneGap, and an HTML viewer to display the SAPUI5 content on the user interface.
  • Using Images in Mobile Applications
    Some notes on using images in mobile applications.
  • Message Handling
    Recommended guidelines for message handling.
  • Mobile Events
    In addition to the general browser and control events you can access specific events for touch-enabled devices.
  • Performance Behavior
    Mobile devices usually have a limited memory and processing power. Complex web pages may therefore have a negative impact on the application performance. This also depends on the mobile device and affects most likely the sap.m.List and sap.m.Table controls.
