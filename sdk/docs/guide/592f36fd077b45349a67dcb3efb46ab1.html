Create Your First Mobile OpenUI5 Application

Whether you develop a OpenUI5 application for desktop or for mobile, the concepts are very similar. OpenUI5 provides an additional control library called sap.m, which is optimized for mobile devices, but also runs fine in desktop browsers.

Note If you need help on choosing the right browser or platform, check here: sap.m.
In this section:
  • Create an HTML Page for Your Mobile App
    You first create an HTML page and define the meta tags, a script tag to load the OpenUI5 libraries and a placeholder for your mobile app.
  • Initialize the Mobile App
    The sap.m library provides a control called App which is meant to be the root control of a mobile application. It provides the initialization of the HTML page, sets some meta tags to ensure an as-native-as-possible look&feel, and can manage different pages and the animations between them.
  • Add Content Pages
    Typical mobile applications are often composed of a number of pages/views/screens between which the user can navigate. You now add two of them to your app.
  • Run Your First Mobile App
    Process how to run your first mobile app.
