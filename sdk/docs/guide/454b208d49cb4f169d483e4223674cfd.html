Fragments

Fragments are light-weight UI parts (UI subtrees) which can be reused, defined similar to views, but do not have any controller or other behavior code involved

In the following sections, you find information on how to use fragments.

In this section:
  • Instantiation of Fragments
    OpenUI5 provides the generic function sap.ui.fragment() to instantiate fragments.
  • Unique IDs
    You can use a unique ID for a fragment that will be used as prefix for all controls in a fragment instance.
  • Dialogs and other Popups as Fragments
    You can use fragments to declaratively define dialogs and other popup controls which are not part of the normal page UI structure.
  • Fragments with Multiple Root Nodes
    XML fragments and JS fragments can have more than one root control.
