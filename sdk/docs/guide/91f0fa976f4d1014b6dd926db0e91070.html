Models and Data Binding in Applications

Models and data binding is used in OpenUI5 applications to connect and synchronize two data or information sources.

The following sections describe the process for data binding, how a model instance is created and assigned to the UI and how the controls on the UI are bound to the model. OpenUI5 provides several binding modes and provides multimodel support.

In this section:
  • Instantiating Models
    The first step when using data binding is to instantiate a model and to provide the data for that model.
  • Assigning the Model to the UI
    The model instance must be assigned to the UI, that is, the framework must know about the model instance, before you can bind controls to this model instance.
  • Defining a Binding Path
    Binding paths address the different properties and lists in a model and define how a node in the hierarchical data tree can be found.
  • Binding Controls to the Model
    To retrieve data from the model for visualization in the UI, the controls have to be bound to the model by means of a binding path. The control itself, all properties of controls, and all multiple aggregations can be bound to a model.
  • Setting the Default Binding Mode
    The default binding mode applies when a model instance is created. You can overwrite the default binding mode after model creation.
  • Multimodel Support
    OpenUI5 supports the use of more than one model for data binding.
