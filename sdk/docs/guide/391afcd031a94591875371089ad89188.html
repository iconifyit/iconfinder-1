Adapting to Platform and Form Factors

The OpenUI5 Mobile library target devices run different operating systems and have very different screen sizes and pixel densities. OpenUI5 does some adaptations automatically. In addition to that, the application can further adapt to the current device.

In this section:
  • Built-in Adaptation of OpenUI5 Mobile
    OpenUI5 mobile supports two options for built-in adaptations: Platform-dependent styling and the SplitApp control.
  • Adapting Automatically to Different Screen Sizes using SplitApp
    SplitApp ensures that your UI automatically adapts to the size available on the respective device.
  • Layouting with FlexBox
    The FlexBox control allows to develop layouts which adjust to the available space and avoid unused space or overflow.
  • Checking the Operating System your Application is Running on
    A platform attribute is added to the HTML tag when running on mobile devices.
  • Options for Further Adaptation
    In addition to the adaptations done automatically by OpenUI5, the application can apply further platform adaptations.
  • Compact Mode for sap.m.controls in Blue Crystal
    The compact mode is a reduced-size design which is available for sap.m controls used in the Blue Crystal theme.
