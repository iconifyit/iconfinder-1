Quality

OpenUI5 supports several testing frameworks, such as QUnit testing and One Page Acceptance tests.

This section describes the exact procedures needed to set up specific testing frameworks for the OpenUI5 environment, as well as some guiding principles and best practices, for example relating to QUnit testing and One Page Acceptance (OPA) tests. Furthermore, this section also contains a list of do's and don'ts and a cookbook for writing OpenUI5 controls.

In this section:
  • QUnit Testing Fundamentals
    QUnit is a powerful, easy-to-use JavaScript unit testing framework. It is used by the jQuery, jQuery UI and jQuery Mobile projects and is capable of testing any generic JavaScript code. It supports asynchronous tests out-of-the-box.
  • QUnit Testing for OpenUI5 Controls
    Comprehensive overview of QUnit testing for controls.
  • One Page Acceptance Tests (OPA5)
    OPA5 is an API for OpenUI5 controls. It hides asynchronicity and eases access to OpenUI5 elements. This makes OPA especially helpful for testing user interactions, integration with OpenUI5, navigation, and data binding.
  • Mock Server
    Introduces how mock server is used to simulate OData backend calls
